# Démarrage avec Qwik

Qwik est un nouveau type de framework qui est [résumable](../concepts/resumable/index.mdx) (pas de JS et pas d'hydratation), construit pour le Edge et [familier pour les développeurs React](../cheat/qwik-react/index.mdx).

- [Stackblitz Qwik + Vite](https://qwik.new)
- [Exemples de terrains de jeux](/examples/reactivity/counter/)

## Prérequis

- [Node.js v16.8](https://nodejs.org/en/download/) ou supérieur
- Votre IDE préféré ([vscode](https://code.visualstudio.com/) est recommandé)
- Commencez par lire [Pensez Qwik](../think-qwik/index.mdx)

## Création d'une appli à l'aide du CLI

La première étape est de créer une application. Qwik est livré avec un CLI qui vous permet de créer un squelette de base d'une application. Nous allons utiliser le CLI pour créer un _starter_ vierge afin que vous puissiez vous familiariser rapidement avec lui. Qwik prend en charge NPM, yarn et pnpm.

### Exécuter le CLI de Qwik dans votre shell

Choisissez le gestionnaire de paquets que vous préférez et exécutez la commande suivante :

```shell
npm create qwik@latest
pnpm create qwik@latest
yarn create qwik
```

Le CLI vous guidera à travers un menu interactif pour définir le nom du projet, sélectionner l'un des _starters_ et vous demandera si vous voulez installer les dépendances.

## Écriture de composants Qwik

Les composants sont les éléments de base des applications Qwik. 

Les [composants Qwik](../components/overview/index.mdx) devraient être familiers à de nombreux développeurs d'interface utilisateur :

```tsx
import { component$, useSignal } from '@builder.io/qwik';

export const MyCmp = component$((props: MyCmpProps) => {
  // Déclare l'état local
  const count = useSignal(0);

  // Retourne le JSX
  return (
    <>
      <h3>Hello, {props.name}</h3>
      <p>Vous avez cliqué {count.value} fois</p>
      <button
        onClick$={() => {
          // Cela mettra à jour l'état local et provoquera un nouveau rendu.
          // La réactivité est au cœur de Qwik !
          count.value++;
        }}
      >
        Incrémenter
      </button>
    </>
  );
});
```

Les composants Qwik sont uniques en cela :

- Les composants Qwik sont automatiquement décomposés en morceaux avec un chargement différé par l'[Optimiseur](../../advanced/optimizer/index.mdx).
- Ils sont [resumables](../../concepts/resumable/index.mdx). (Un composant peut être créé sur un serveur et continuer son exécution sur le client).
- Ils sont [réactifs](../../concepts/reactivity/index.mdx)  et peuvent effectuer un rendu indépendamment des autres composants de la page. Consultez le [rendu](../../components/rendering/index.mdx).

Apprenez-en plus sur les concepts clés de l'écriture des [Composants Qwik](../components/overview/index.mdx).
