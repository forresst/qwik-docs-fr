# Structure du projet

Un projet typique de Qwik ressemble à ceci :

```bash
qwik-app-demo
├── README.md
├── package.json
├── public
│   └── favicon.svg
├── src
│   ├── components
│   │   └── router-head
│   │       └── router-head.tsx
│   ├── entry.ssr.tsx
│   ├── global.css
│   ├── root.tsx
│   └── routes
│       ├── flower
│       │   ├── flower.css
│       │   └── index.tsx
│       ├── index.tsx
│       ├── layout.tsx
│       └── service-worker.ts
├── tsconfig.json
└── vite.config.ts
```

## Fichiers du projet

### `src/routes/`

Le répertoire `src/routes/` est un répertoire spécial pour Qwik City, puisque c'est le répertoire où Qwik City va chercher vos pages. Les dossiers et fichiers à l'intérieur de ce répertoire ont une signification particulière et ils seront mappés à l'URL de votre appli.

- `src/routes/index.tsx` est la page d'accueil de votre appli.
- `src/routes/layout.tsx` est la mise en page racine de votre appli, toutes les pages seront rendues dans cette mise en page.

> Reportez-vous à la section [Routage](/qwikcity/routing) pour plus d'informations.

### `src/components/`

Le répertoire `src/components/` est un répertoire par convention. Tous les starters Qwik auront ce répertoire, mais vous pouvez le changer si vous le souhaitez. Ce répertoire est l'endroit où vous devriez mettre vos composants, c'est à dire des morceaux de code réutilisables qui peuvent être utilisés à plusieurs endroits. Ce ne sont pas des routes ou des layouts, mais ils peuvent être utilisés à l'intérieur de ceux-ci.

Par exemple, un composant `Button` doit se trouver dans `src/components/button/button.tsx`.

### `public/`

Le répertoire `public/` contient des ressources statiques telles que des images, des polices et des icônes. Lorsque vous construisez votre application, ces fichiers seront copiés dans le répertoire `dist/` et seront servis à la racine.

Reportez-vous à [Configuration de Vite pour plus d'informations](https://vitejs.dev/guide/build.html#public-base-path).

### `src/entry.ssr.tsx`

Le point d'entrée SSR, dans tous les cas où l'application est rendue en dehors du navigateur, ce point d'entrée sera le point commun.

- Server (express, cloudflare...)
- npm run start
- npm run preview
- npm run build

### `src/root.tsx`

Le fichier `src/root.tsx` est le point d'entrée de l'arbre de l'application. C'est le premier composant qui sera rendu. C'est la racine de l'arbre.

### `src/global.css`

Le fichier `src/global.css` est un fichier CSS global, si vous avez besoin de définir un CSS qui est utilisé à plusieurs endroits dans votre application, vous pouvez le définir ici.

Le composant racine (`src/root.tsx`) importe ce fichier par défaut.

### `tsconfig.json`

Le fichier `tsconfig.json` contient la configuration du compilateur TypeScript. Il est standard pour tout projet TypeScript.

### `vite.config.ts`

Qwik utilise Vite pour construire le projet. Le fichier `vite.config.ts` contient la configuration de Vite. C'est un standard pour tout projet Vite. Veuillez vous référer à la [documentation Vite](https://vitejs.dev/config/) pour plus d'informations.
