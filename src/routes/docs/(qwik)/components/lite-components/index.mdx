# Composants légers

En plus des `component$()` standards de Qwik qui ont toutes les propriétés de chargement
différé, Qwik prend aussi en charge les composants de faible poids (léger) qui se comportent plus
comme dans des frameworks traditionnels. Les composants légers peuvent également être appelés
composants en ligne.

```tsx
import { component$ } from '@builder.io/qwik';

// Composant léger : déclaré à l'aide d'une fonction standard.
export const MyButton = (props: { text: string }) => {
  return <button>{props.text}</button>;
};

// Composant : déclaré en utilisant `component$()`.
export const MyApp = component$(() => {
  return (
    <div>
      Du texte :
      <MyButton text="Cliquez-moi" />
    </div>
  );
});
```

Dans l'exemple ci-dessus, le `MyButton` est un composant léger.
Contrairement au `component$()` standard, les composants légers ne peuvent pas être chargés individuellement
de manière différée; ils sont regroupés avec leur composant parent. Dans ce cas :

- `MyButton` sera mis dans un bundle avec le composant `MyApp`.
- Chaque fois que `MyApp` sera rendu, il garantira également que `MyButton` soit
rendu.
- `MyButton` n'a pas d'[élément hôte](../../../integrations/integration/react/#élément-hôte).

On peut considérer que les composants légers sont intégrés dans le composant où ils sont instanciés.

## Limitations
Les composants légers ont certaines limitations que le `component$()` standard
n'a pas. standard. Les composants légers :
- Ne peuvent pas utiliser les méthodes `use` telles que `useSignal` ou `useStore`.

Comme leur nom l'indique, les composants légers sont à utiliser avec parcimonie
pour des éléments de balisage légers, car ils offrent l'avantage d'être
faciles à regrouper avec le composant parent.
