Utilisez le contexte pour transmettre des données aux composants enfants sans les transmettre explicitement à travers les composants (connu sous le nom de prop-drilling). Le contexte est utile pour partager des données qui sont nécessaires dans tous les composants de l'application. Par exemple, les informations de style, l'état de l'application ou l'utilisateur actuellement connecté.

Pour utiliser le contexte, vous avez besoin de trois éléments :

- `createContextId()` : ceci crée un ID sérialisable pour le contexte. Assurez-vous que cet identifiant est unique dans votre application.
- `useContextProvider()` : Au niveau d'un composant parent, appelez cette méthode pour publier la valeur du contexte. Tous les enfants (et petits-enfants) qui sont des descendants de ce composant seront en mesure de récupérer le contexte.
- [`useContext()`](/docs/components/context/index.mdx#usecontext) pour récupérer le contexte et l'utiliser dans n'importe quel composant.

Dans cet exemple, nous souhaitons passer le `TodosStore` au composant `<Items>`. Mettez à jour le code pour utiliser [`useContext()`](/docs/components/context/index.mdx#usecontext) pour récupérer la valeur.

Les contextes sont généralement des stores, et en tant que tels, ils doivent être sérialisables.
