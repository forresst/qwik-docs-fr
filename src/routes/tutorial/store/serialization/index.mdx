Les stores sont sérialisables lorsque l'application est mise en pause sur le serveur et reprise sur le client.

Les données dans un store n'ont pas besoin d'être libre de références imbriquées. Qwik est parfaitement capable de sérialiser les stores qui contiennent [cycles](https://wikipedia.org/wiki/Directed_acyclic_graph) de données.

Les stores peuvent aussi inclure des [URL Qwik (QRL)](../../../docs/(qwik)/advanced/qrl/index.mdx).

Il n'y a pas de limite au nombre de stores qui peuvent être créés par composant, ni à l'endroit où ils sont passés. Alors que la meilleure pratique est d'implémenter des flux de données descendants, Qwik n'exige pas cette philosophie de conception.

> **Votre tâche** : Insérez quelques éléments dans `parent.children` pour créer une référence circulaire entre les stores parent et enfant.

Cet exemple montre aussi comment attacher et appeler des closures à un store en utilisant `$` et `QRL`.
