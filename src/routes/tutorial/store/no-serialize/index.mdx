Les stores doivent toujours être sérialisables, mais il peut arriver que vous souhaitiez faire persister une valeur sur un store qui ne peut pas être sérialisé. Les valeurs de ce type ne persistent dans le navigateur que lorsque l'application est reprise sur le client.

Lorsque l'application est mise en pause, les propriétés non sérialisables sont éliminées.

> **Votre tâche** : Ajoutez une fonction de nettoyage dans `store.cleanup` pour que le bouton `stop` puisse effacer l'intervalle avec `clearInterval()`.
> Pour cela, ajoutez `noSerialize` importé de '@builder.io/qwik' et utilisez la fonction `noSerialize` pour envelopper la fonction clearInterval.
