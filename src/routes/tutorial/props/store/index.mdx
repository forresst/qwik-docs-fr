Le tutoriel précédent vous a présenté la manière dont vous pouviez transmettre des primitives et des objets aux composants. Dans ce tutoriel, nous allons apprendre à passer des stores aux composants, ce qui est beaucoup plus efficace en termes de rendu.

Lorsque vous cliquez sur le bouton `+1`, le rendu de `<App>` est modifié pour mettre à jour les liaisons de `<Display>`. Le rendu `<App>` est nécessaire pour mettre à jour les props de `<Display>`, mais ce processus ne correspond pas à ce que l'utilisateur voit, c'est donc un gaspillage de ressources.

Une meilleure approche consiste à ne refaire que le rendu du composant `<Display>` en passant la référence `CountStore` plutôt que la valeur `count`. Comme la référence au store ne change jamais, le composant `<App>` n'a pas besoin d'être re-rendu.

> **Votre tâche** : Changez le code pour passer dans `store` plutôt que `store.count`.

La modification ci-dessus présente deux avantages :

- Le composant `<App>` n'a pas besoin d'être chargé ou re-rendu.

## Bonne pratique

Une bonne pratique à suivre dans Qwik est de passer un store aux composants enfants plutôt que des primitives individuelles. Lorsque vous utilisez des primitives, le composant parent doit effectuer un nouveau rendu juste pour transmettre la nouvelle valeur. Lorsque vous passez par un store, le composant parent n'a pas besoin de faire un re-rendu.
