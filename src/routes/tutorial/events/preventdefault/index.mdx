Pour certains événements, les navigateurs ont un comportement par défaut. Par exemple, lorsque vous cliquez sur un lien, le navigateur va généralement naviguer vers le `href` du lien. Il arrive que l'on veuille empêcher cela. C'est pourquoi l'API `Event` contient la méthode `preventDefault()`.

Les événements du navigateur sont synchrones, mais comme Qwik est un chargeur à grain fin, le modèle d'exécution de Qwik est asynchrone. Cela signifie qu'au moment où l'événement est déclenché, le gestionnaire d'événement n'est pas encore chargé. Au moment où l'événement est chargé, l'événement a déjà été traité par le navigateur et appeler `preventDefault()` n'aura aucun effet.

Pour résoudre ce problème, Qwik fournit une API déclarative pour appeler automatiquement `preventDefault()` quand l'événement est déclenché. Ceci est réalisé en ajoutant l'attribut `preventdefault:<event-name>` à l'élément. Cela permet au Qwikloader d'appeler de manière synchrone `preventDefault()` lorsque l'événement est déclenché.

### Exemple

Le fait de cliquer sur le lien provoquera un événement de navigation. Nous souhaitons empêcher cela et appeler notre callback à la place. Ajoutez `preventdefault:click` au `<a href>` pour y parvenir.
