Il arrive que votre appli ait besoin d'ajouter un écouteur conditionnel. Parfois le nom d'un écouteur est inconnu, ou vous devez utiliser une approche impérative pour configurer un écouteur.

Qwik fournit les fonctions suivantes pour attacher un écouteur :

| Fonction          | Description                                                                |
| ----------------- | -------------------------------------------------------------------------- |
| `useOn()`         | Ajoute un événement sur un événement spécifique sur l'élément hôte actuel. |
| `useOnDocument()` | Ajoute un événement sur un événement spécifique sur document.              |
| `useOnWindow()`   | Ajoute un événement sur un événement spécifique sur window.                |

> **Votre tâche** : Configurez un écouteur de clic sur le composant pour appeler `alert('Hello world!')`.

## Comprendre le `$`

La fonction `$` signale à Qwik de charger de manière différée une référence. Lors de la mise en place d'un écouteur avec `useOn`, le second argument est une [URL Qwik](../../../docs/advanced/qrl/index.mdx).

Les URLs Qwik (QRL) sont des références chargées de manière différée sur votre code. Si `useOn` prenait une fonction directement plutôt qu'une QRL, il devrait s'exécuter par avance pour allouer la closure de l'écouteur. En utilisant une QRL via la fonction `$`, Qwik peut charger de manière différée la closure seulement quand l'écouteur `click` est déclenché.
