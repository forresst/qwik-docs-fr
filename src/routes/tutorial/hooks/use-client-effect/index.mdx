Utilisez `useClientEffect$()` pour exécuter du code après que le composant soit résumé. Ceci est utile pour mettre en place des temporisateurs ou des flux sur le client lorsque l'application est résumée.

## `track()` et `useClientEffect$()`

`useClientEffect$()` reçoit une fonction `track()` tout comme `useWatch$()`. Utilisez la fonction `track()` pour déclencher l'effet lorsque le store est mis à jour. Consultez la section [réactivité](../../../tutorial/reactivity/explicit/index.mdx) pour plus d'informations.

## Cycle de vie des composants et SSR

Qwik est résumable. La résumabilité signifie que l'application démarre sur le serveur et est ensuite transférée sur le client. Sur le client, l'application continue l'exécution là où elle s'est arrêtée. Un cas d'utilisation commun est de créer un composant sur le serveur, de le mettre en pause, puis de le reprendre sur le client. Pour que le composant soit pleinement fonctionnel, il peut être nécessaire d'exécuter du code par avance sur le client pour configurer des temporisateurs ou des flux.

`useClientEffect$()` est une méthode réservée au client. (Il n'y a pas d'équivalent sur le serveur).

> **REMARQUE** Consultez [`useWatch$()`](/docs/components/lifecycle/index.mdx#usewatch) pour les comportements qui doivent être exécutés à la fois sur le client et le serveur.

## Quand est-ce que `useClientEffect$()` est exécuté ?

Le code de l'effet client est exécuté après que le composant a été résumé. La méthode `useClientEffect$()` prend un argument supplémentaire (`{eagerness:'visible|load|idle'}`) qui contrôle quand l'effet est exécuté. Il existe trois options :
- `visible` (par défaut) : Exécute l'effet lorsque le composant devient visible. Cette option est préférable car elle retarde l'exécution de l'effet jusqu'à ce que le composant soit visible plutôt que de l'exécuter par avance au démarrage de l'application (nous essayons de minimiser la quantité de code que l'application exécute au démarrage).
- `load` : Exécute le code dès que possible. C'est généralement juste après l'événement `DOMContentLoaded`.
- `idle` : Exécuter le code lorsque le prochain callback de l'idle est déclenché.

### Exemple

L'exemple montre un composant horloge qui est rendu en dessous du cadre. Utilisez la fonction `useClientEffect$()` pour que l'horloge mette à jour l'heure actuelle toutes les secondes afin qu'elle fonctionne sur le client. Nous avons fourni la fonction utilitaire `updateClock` pour faciliter votre implémentation.

Gardez à l'esprit que `useClientEffect$()` doit retourner une fonction de nettoyage qui libère le temporisateur `setInterval` afin que le temporisateur puisse être correctement nettoyé lorsque le composant est démonté/détruit.
