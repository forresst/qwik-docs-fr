Utilisez `useMount$()` pour exécuter du code lorsque le composant est monté dans l'arborescence de rendu. (Une autre façon d'y penser est que `useMount$()` s'exécute à la création du composant).

## Cycle de vie des composants et SSR

Qwik est résumable. La résumabilité signifie que l'application démarre sur le serveur et ensuite l'application est transférée sur le client. Sur le client, l'application continue l'exécution là où elle s'est arrêtée. L'implication de ceci est qu'un composant peut être créé sur le serveur et détruit sur le client. Cela signifie que la méthode `useMount$()` du composant peut s'exécuter sur le serveur alors que sa méthode `useCleanup$()` peut s'exécuter sur le client.

## Utilisation de `useMount$()`

`useMount$()` est un hook qui exécute un callback lorsque le composant est monté dans l'arborescence de rendu. La fonction `useMount$()` peut être asynchrone. `useMount$()` retarde le rendu du composant jusqu'à ce que le callback `useMount$()` ait fini de s'exécuter. L'utilisation typique de `useMount$()` est de récupérer les données nécessaires au rendu initial.

## Variantes de `useMount$()`

Le hook `useMount$()` est divisé en deux versions, l'une serveur et l'autre client :

- `useServerMount$()` : Hook qui s'exécute sur le montage du composant lorsqu'il se trouve dans un environnement serveur. Ceci est utile car le serveur a souvent des API différentes pour la récupération des données.

Utilisez `useMount$()` si le code qui doit être exécuté est identique.

## Importations sur serveur uniquement

Comme les hooks `useServerMount$()` ont `$` dans leur nom, ils sont sujets au chargement différé. Le chargement différé signifie que la fonction est déplacée dans un nouveau fichier par l'Optimiseur. Lorsque la fonction est déplacée, l'Optimiseur déplace également toutes les importations avec elle. Cela signifie qu'il n'y a pas de danger à avoir des imports réservés au serveur dans le hook `useServerMount$()` car ils seront supprimés par l'Optimiseur.

Utilisez `useServerMount()` pour récupérer les données nécessaires au rendu.
