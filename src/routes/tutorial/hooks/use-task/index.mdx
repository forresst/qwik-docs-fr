Utilisez [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask) pour exécuter une fonction avant le rendu initial et à chaque fois que les valeurs traquées changent. La fonction s'exécute avant le rendu, mais elle ne peut pas retarder le rendu, donc si [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask) est asynchrone, le rendu aura lieu avant que [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask) soit complètement exécuté.

## Suivi des modifications apportées aux stores

Le hook [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask) reçoit une fonction `track()` utilisée pour configurer les abonnements qui relanceront automatiquement le hook [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask). L'ensemble des propriétés de `track()` se réinitialise à chaque exécution de [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask), il est donc important de toujours utiliser `track()` pour re-configurer les abonnements. Pour cette raison, il est possible que le [`useTask$()`](/docs/(qwik)/components/lifecycle/index.mdx#usetask) s'abonne à différentes propriétés au fil du temps.

## Nettoyage
