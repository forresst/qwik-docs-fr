Dans les cas simples, la projection permet au contenu du composant parent d'être projeté dans le composant enfant. Dans des cas plus complexes, il peut y avoir plus d'un emplacement de contenu qui doit être projeté. Pour disposer de plusieurs emplacements de contenu, il suffit de leur donner un nom.

### Exemple

Dans cet exemple, nous avons créé un composant `<Collapsable>` qui bascule entre les états ouvert et fermé. Actuellement, lorsque le composant `<Collapsable>` est fermé, il affiche le contenu qui a été implémenté dans le composant `<Collapsable>`. Remplacez cela par un `<Slot>` supplémentaire pour projeter le contenu `q:slot="closed"` du parent à la place.

## Slots non projetés

Nous avons ajouté des instructions de console pour montrer quand les composants individuels sont re-rendus. Remarquez que le composant `<App>` ne se re-rend jamais sur le client. Remarquez également que le composant `<Collapsable>` ne projette qu'un seul contenu à la fois. Cela signifie que lorsque `<App>` se rend sur le serveur, il doit produire à la fois le contenu par défaut ainsi que le contenu `closed` que Qwik doit sérialiser. L'avantage est que lorsque `<Collapsable>` bascule entre les états ouvert et fermé, il n'a pas besoin de refaire le rendu du composant `<App>` pour récupérer le contenu qui a été projeté dedans.
