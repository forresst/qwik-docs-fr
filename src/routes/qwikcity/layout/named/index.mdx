# Mise en page nommée

Il arrive que des routes connexes doivent avoir des mises en page radicalement différentes de celles de leurs voisines. Il est possible de définir plusieurs mises en page pour des routes voisines. Une seule mise en page par défaut et un nombre quelconque de mises en page nommées. La route enfant peut alors demander une mise en page spécifique nommée.

Qwik City définit la convention pour que les mises en page soient dans `src/routes` et que le nom du fichier commence par `layout`. C'est pourquoi le layout par défaut est nommé `layout.tsx`. Un layout nommé commence aussi par `layout` suivi d'un tiret `-` et d'un nom unique, comme `layout-narrow.tsx`.

```
src/
└── routes/
    ├── contact/
    │   └── index@narrow.tsx      # https://example.com/contact (Mise en page : layout-narrow.tsx)
    ├── layout.tsx                # Mise en page par défaut
    ├── layout-narrow.tsx         # Mise en page nommée
    └── index.tsx                 # https://example.com/ (Mise en page : layout.tsx)
```

- `https://example.com/`
  ```
  ┌──────────────────────────────────────────────────┐
  │       src/routes/layout.tsx                      │
  │  ┌────────────────────────────────────────────┐  │
  │  │    src/routes/index.tsx                    │  │
  │  │                                            │  │
  │  └────────────────────────────────────────────┘  │
  │                                                  │
  └──────────────────────────────────────────────────┘
  ```
- `https://example.com/contact`
  ```
  ┌──────────────────────────────────────────────────┐
  │       src/routes/layout-narrow.tsx               │
  │  ┌────────────────────────────────────────────┐  │
  │  │    src/routes/contact/index@narrow.tsx     │  │
  │  │                                            │  │
  │  └────────────────────────────────────────────┘  │
  │                                                  │
  └──────────────────────────────────────────────────┘
  ```
