# Mises en page groupées

Les routes utilisées à des fins communes sont souvent placées dans des répertoires afin qu'elles puissent partager des mises en page, et que les fichiers sources apparentés soient logiquement regroupés les uns à côté des autres. Cependant, il peut être souhaitable que le répertoire, qui a été utilisé pour regrouper des fichiers similaires et partager des présentations, soit exclu de l'URL publique. C'est là qu'interviennent les mises en page "groupées" (également appelées routes de mise en page « sans chemin »).

En entourant tout nom de répertoire de parenthèses, comme `(nom)`, le nom du répertoire lui-même ne sera pas inclus dans le chemin d'accès de l'URL.

Par exemple, disons qu'une appli a placé toutes les routes _account_ dans un répertoire, cependant, `/account/` pourrait être supprimé de l'URL pour des URL plus propres et plus courtes. Dans l'exemple ci-dessous, remarquez que les chemins sont dans le répertoire `src/routes/(account)/`. Cependant, les chemins de l'URL excluent `(account)/`.

```
src/
└── routes/
    └── (account)              # Remarquez les parenthèses
        ├── layout.tsx         # Mise en page partagée du compte
        └── profile/
            └── index.tsx      # https://example.com/profile
        └── settings/
            └── index.tsx      # https://example.com/settings
```
