# Disposition des fichiers

La responsabilité principale de Qwik City est de fournir un moyen simple, intuitif et efficace de créer du contenu pour votre site. Qwik City réalise cela grâce au routage basé sur les répertoires. En un mot, la disposition de vos répertoires détermine la structure de votre site.

## Le répertoire `routes`

Toutes les informations sur les routes basées sur les répertoires sont placées dans le répertoire `src/routes`. C'est le seul répertoire où les conventions de nommage des répertoires et des fichiers ont des implications pour votre site. (Dans tous les autres répertoires, les noms n'ont aucune signification pour Qwik City).

(le dossier `src` contient aussi typiquement des `components`, mais le nom et l'emplacement n'ont aucune signification pour Qwik City).

## Exemple du répertoire `routes`

```
──src/
  ├─components/          # Mettez ici vos composants réutilisables.
  │ └─counter.tsx
  └─routes/              # Mettez ici vos composants spécifiques aux routes.
    ├─docs/
    │ ├─overview/
    │ │ └─index.mdx      # https://example.com/docs/overview
    │ └─index.mdx        # https://example.com/docs
    ├─index.tsx          # https://example.com/
    └─layout.tsx         # Toutes les pages imbriquées utilisent cette mise en page
```

## Exemple de composant

Qwik City s'attend à ce que les fichiers `index.tsx` qui sont dans le dossier `src/routes` aient un export `default`.

```tsx
import { component$ } from '@builder.io/qwik';

export default component$(() => {
  return <>Hello World!</>;
});
```
