# `loader$()`

Les chargeurs permettent aux données de circuler du serveur vers les composants Qwik. Pour cette raison, les chargeurs Qwik doivent toujours être inclus dans le contexte du **chargement de données sur le serveur qui sont ensuite consommées par un composant Qwik**, si vous voulez créer un endpoint RESTful, veuillez consulter à la place le guide [endpoints](/qwikcity/endpoints/index.mdx).

Ils se comportent comme des fonctions RPC côté serveur qui peuvent être appelées par les composants Qwik pendant le rendu.

## Déclaration d'un chargeur

Les chargeurs ne peuvent être déclarés que dans le dossier `src/routes`, dans un fichier `layout.tsx` ou `index.tsx`, et ils DOIVENT être exportés.

```tsx
// src/routes/layout.tsx
import { loader$ } from '@builder.io/qwik-city';

export const getProductData = loader$(() => {
  return {
    product: {
      name: 'Qwik City',
      price: 100,
    },
  };
});
```

> Les loaders ne sont pas des endpoints, c'est un canal de communication interne entre le serveur et les composants Qwik. Aussi, les chargeurs doivent retourner une valeur JSON sérialisable.

## Utilisation d'un chargeur

Les chargeurs peuvent être utilisés par n'importe quel composant de l'application, tant que le chargeur est déclaré dans un fichier `layout.tsx` ou `index.tsx` qui fait partie de la route existante.

```tsx
// src/routes/index.tsx
import { loader$ } from '@builder.io/qwik-city';
import { component$ } from '@builder.io/qwik';

export const useGetServerTime = loader$(() => {
  return {
    time: Date.now();
  }
});

export default component$(() => {
  // Récupére un signal réactif des données du chargeur
  const signal = useGetServerTime(); // Signal<{time: number}>
  return (
    <div>
      Heure du serveur : {signal.value.time}
    </div>
  );
});
```

> La fonction `useGetServerTime()` récupère les données du chargeur, mais ne l'exécute pas plusieurs fois. Les chargeurs s'exécutent par avance au début de la requête afin d'offrir une faible latence. Pour cette raison, ils ne sont autorisés que dans le dossier `src/routes`, dans un fichier `layout.tsx` ou `index.tsx`, et ils DOIVENT être exportés.

## Plusieurs chargeurs

Plusieurs chargeurs sont autorisés dans toute l'application, et ils peuvent être utilisés dans n'importe quel composant Qwik. **Vous pouvez même déclarer plusieurs chargeurs dans le même fichier**.

**Fichier : `src/routes/layout.tsx`**

```tsx
import { component$ } from '@builder.io/qwik';
import { loader$ } from '@builder.io/qwik-city';
import { Footer } from '../components/footer.tsx';

export const useGetServerTime = loader$(() => {
  return {
    time: Date.now();
  }
});

export default component$(() => {
  const signal = useGetServerTime();
  return (
    <main>
      <Slot>
      <Footer />
    </main>
  );
});
```

**Fichier : `src/components/footer.tsx`**

```tsx
import { component$ } from '@builder.io/qwik';

// Importe le chargeur à partir du layout
import { useGetServerTime } from '../routes/layout.tsx';

export const Footer = component$(() => {
  // Consomme les données du chargeur
  const signal = useGetServerTime();
  return <footer>Server time: {signal.value.time}</footer>;
});
```

**Fichier : `src/routes/admin/index.tsx`**

```tsx
import { component$ } from '@builder.io/qwik';
import { loader$ } from '@builder.io/qwik-city';

export const useIsLogged = loader$(({cookie}) => {
  return {
    logged: checkcookie(cookie);
  }
});

export const useGetCurrentUser = loader$(({cookie}) => {
  return {
    user: currentUserFromcookie(cookie);
  }
});

export default component$(() => {
  const logged = useIsLogged();
  const user = useGetCurrentUser();
  return (
    <section>
      <h1>Admin</h1>
      {logged.value.logged ? (
        <p>Bienvenue {user.value.user.name}</p>
      ) : (
        <p>Vous n'êtes pas connecté</p>
      )}
    </section>
  );
});
```

## Contexte du chargeur

Tout comme les gestionnaires de requêtes tels que `onRequest` et `onGet`, les chargeurs ont accès à l'objet `RequestEvent` qui contient des informations sur la requête en cours.

Ces informations sont très utiles lorsque le chargeur doit retourner des données de manière conditionnelle en fonction de la requête, ou lorsqu'il doit modifier manuellement le statut, les entêtes ou le corps de la réponse.

```tsx
import { loader$ } from '@builder.io/qwik-city';

export const getServerTime = loader$((ev) => {
  console.log('Request headers:', ev.headers);
  console.log('Request cookies:', ev.cookie);
  console.log('Request url:', ev.url);
  console.log('Request method:', ev.method);
  console.log('Request params:', ev.params);

  return {
    time: Date.now();
  }
});
```

## Quand utiliser un chargeur ?

Un chargeur doit être utilisé lorsque vous avez besoin de fournir des données côté serveur à vos composants Qwik. Par exemple, si vous avez besoin de récupérer des données d'une base de données ou d'une API, vous pouvez utiliser un chargeur pour le faire.

Vous ne devriez pas utiliser un chargeur pour créer une API REST, pour cela vous feriez mieux d'utiliser un [Endpoint](/qwikcity/endpoints/index.mdx), qui vous permet d'avoir un contrôle précis sur les entêtes et le corps de la réponse.
